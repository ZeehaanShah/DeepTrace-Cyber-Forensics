## Overview

Cyber threats in 2025 are not just technical — they are linguistic. Phishing emails, scam messages, and AI-generated disinformation exploit human psychology, not software vulnerabilities. DeepTrace addresses this by embedding forensic analysis capabilities directly into the browser, where threats are encountered.

**What DeepTrace does in real time:**

- Analyzes any URL for phishing signals using a 52-feature engineered ML ensemble
- Classifies pasted text as phishing/scam using a fine-tuned DeBERTa transformer
- Detects whether text was written by a human or generated by an AI (GPT, etc.) using a pretrained RoBERTa model
- Combines all three signals into a single verdict via a logistic regression meta-classifier
- Provides explainable output: confidence scores, risk levels, triggered indicators, and actionable suggestions

## Detection Modules

### Module 1 — Phishing URL Detection

**Architecture:** XGBoost + LightGBM ensemble (50/50 weight average)

**52 Engineered Features across 6 categories:**

| Category | Features |
|---|---|
| Length-based | URL length, domain length, subdomain length, path length, query length |
| Character counts | Dots, hyphens, slashes, `@`, `?`, `=`, `%`, `+`, `~`, `#`, `$`, `&` |
| Statistical | Digit ratio, alpha ratio, domain entropy, full URL entropy |
| Structural flags | IP address as domain, `@` in URL, non-HTTPS, nested subdomains, URL encoding |
| Keyword signals | Phishing keywords (login/verify/secure), spam keywords (free/win/prize), executable extensions |
| Brand/domain | Brand impersonation, URL shorteners, punycode, suspicious TLDs (`.tk`, `.ml`, `.ga`) |

**Triggered indicators shown to user (examples):**
- IP address used as domain
- Brand impersonation detected
- Phishing keywords (login/verify)
- URL shortener detected
- Suspicious TLD

---

### Module 2 — Phishing Text Detection

**Architecture:** Fine-tuned `microsoft/deberta-v3-xsmall` transformer + rule-based pattern boost

**Training:** Binary classification (Legitimate: 0, Phishing: 1)

**9 Rule-based Pattern Detectors (applied as score boost):**
1. Click-bait verification links
2. Account threat language (`suspended`, `expired`, `locked`)
3. Prize/lottery scam phrases
4. Urgency manipulation (`urgent`, `act now`)
5. Payment requests (money, bitcoin, gift card)
6. OTP phishing patterns
7. Generic mass-phish salutations (`Dear Customer`)
8. Time pressure tactics
9. Credential harvesting attempts

**Risky keyword detection:** 25+ flagged terms including `urgent`, `OTP`, `verify`, `bitcoin`, `gift card`, `unauthorized`

---

### Module 3 — AI-Generated Text Detection

**Architecture:** Pretrained `Hello-SimpleAI/chatgpt-detector-roberta` (RoBERTa-base)

**Dataset:** HC3 — Human ChatGPT Comparison Corpus (~27,000 QA pairs)

**Label Mapping:**
- Index 0 → Human-written
- Index 1 → AI-generated (ChatGPT/LLM)

**Minimum input:** 50 characters. Maximum: 512 tokens (truncated if longer).

**Output:** Human% / AI% visualization with confidence and explanation.

---

### Meta Decision Engine

**Architecture:** Logistic Regression (scikit-learn) with StandardScaler

**5 Input Features:**
1. `url_phish_score` — from Module 1
2. `text_phish_score` — from Module 2
3. `ai_detect_score` — from Module 3
4. `url_text_diff` — `|url_score - text_score|`
5. `combined_risk` — `(url_score + text_score) / 2`

**Key Engineering Decisions:**
- Skipped modules receive `0.5` (neutral), not `0.0` — prevents false-legitimate bias
- If meta says "legitimate" but any module score ≥ 0.75 (phishing) or ≥ 0.80 (AI), the high-confidence module overrides the meta verdict
- Falls back to rule-based decisions for single-module runs

## Getting Started

### Prerequisites

- Python 3.11+
- pip
- Google Chrome (latest)
- (Optional) Docker

### Backend Setup

**Step 1 — Clone the repository**

```bash
git clone https://github.com/YOUR_USERNAME/DeepTrace.git
cd DeepTrace
```

**Step 2 — Create a virtual environment (recommended)**

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python -m venv venv
source venv/bin/activate
```

**Step 3 — Install dependencies**

```bash
cd backend
pip install -r requirements.txt
```

**Step 4 — Start the backend server**

```bash
uvicorn app.main:app --reload --port 8000
```

**Expected startup output:**

```
INFO  | DeepTrace starting — loading models...
INFO  | URL models loaded — AUC=0.9885 F1=0.9539 threshold=0.520
INFO  | Text model loaded on cpu — F1=0.9801
INFO  | AI detector loaded on cpu — pretrained RoBERTa ~97% accuracy
INFO  | Meta-classifier loaded
INFO  | Application startup complete.
```

**Step 5 — Verify the backend**

```bash
# Health check
curl http://localhost:8000/health

# Interactive API docs
# Open: http://localhost:8000/docs
```

### Chrome Extension Setup

1. Open Chrome and navigate to `chrome://extensions/`
2. Enable **Developer mode** (top-right toggle)
3. Click **Load unpacked**
4. Select the `extension/` folder inside DeepTrace (the folder containing `manifest.json`)
5. Pin DeepTrace from the extensions puzzle icon

> **Note:** The backend must be running on `localhost:8000` before using the extension.

### POST `/analyze`

**Request Body**

```json
{
  "text": "Dear customer, your account has been suspended. Verify your OTP immediately.",
  "url": "http://paypa1-secure.xyz/verify",
  "source": "manual",
  "mode": "both"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `text` | string | One of text/url | Text to analyze (max 5000 chars) |
| `url` | string | One of text/url | URL to analyze (max 2048 chars) |
| `source` | string | No | Origin identifier |
| `mode` | string | No | `url`, `text`, `ai`, `text_ai`, `both` |

**Analysis Modes**

| Mode | Modules Active | Use Case |
|---|---|---|
| `url` | URL only | Analyze a link |
| `text` | Text phishing only | Check a message for scam |
| `ai` | AI detection only | Check if text is AI-written |
| `text_ai` | Text + AI | Full text forensics |
| `both` | All three | Complete analysis (default) |

**Example Response**
